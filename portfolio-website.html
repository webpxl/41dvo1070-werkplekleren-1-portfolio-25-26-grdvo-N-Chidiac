## Portfolio Website – “Je kijkt er nu naar”

**Vak:** Werkplekleren 1

**Jaar:** 2025-2026

**Live:** https://nourchidiac.netlify.app/

### Opdracht in één zin

Bouw een volledig functioneel, responsive portfolio met **alleen HTML, CSS en plain JavaScript** dat jouw persoonlijkheid en vaardigheden toont – en wees vooral creatief.

### Wat ik precies moest doen

Een persoonlijke website maken met minstens:

  • korte voorstelling
  • motivatie voor de opleiding
  • projecten (min. 3 school + niet-schoolprojecten)
  • minstens 1 “speciale” sectie
  • aparte /portfolio-map met situering + reflectie
  • Alles clean coderen, semantic HTML, nette mappenstructuur
  • Responsive voor mobiel, tablet én desktop
  • GitHub verplicht (min. 5 commits) + publiceren via GitHub Pages
  • Privacy-link, rechtenvrije beelden, bronnen vermelden in README

### Mijn aanpak & keuzes

Ik heb mijn volledige portfolio opgebouwd als een **openstaand VS Code-venster** – precies het programma waarin ik deze site ook heb geschreven. Sidebar met bestanden, tab-balk, statusbar, syntax-highlighting-stijl,... alles verwijst naar de tool die voor mij “thuis” voelt.

Waarom?

Omdat ik geen designer wil worden die alleen maar mooie plaatjes maakt. Ik wil de persoon zijn die die plaatjes laat léven met code. En wat is een betere manier om dat te tonen dan mijn portfolio te laten lijken op het programma waar ik elke dag in zit?

### Geleerd & trots op

  • Voor het eerst een volledige site bouwen zonder één framework of library
  • Responsive maken met alleen flexbox en een paar media queries
  • Een thema maken dat 1-op-1 de look & feel van VS Code nabootst (kleurcodes, font-sizes, alles)
  • Clean commits en een nette README met bronnen (zoals gevraagd)

### Extraatje

De opdracht zelf staat er ook gewoon tussen als project – maar eerlijk. Je kijkt er op dit moment naar.

> “The best way to show you can code… is to code your portfolio like your favorite code editor.”
